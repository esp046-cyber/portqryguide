Complete Beginners Guide to Network Port Testing

üìò What is PortQry?

PortQry is a command-line tool from Microsoft that helps you troubleshoot TCP/IP connectivity issues. It's essential for network administrators and IT professionals in 2026.

Key Features

  • Query TCP and UDP ports on remote computers
  • Check port status (listening, filtered, not listening)
  • Support for multiple protocols (DNS, LDAP, RPC, SQL, etc.)
  • No installation required - portable executable
  • Works on Windows Server and Desktop OS

Installation (2026)

# Download from Microsoft (Free)
https://www.microsoft.com/download/portqry

# Or use Winget (Windows Package Manager)
winget install Microsoft.PortQry

# Extract and run from any folder
portqry.exe -?
üí° Note: PortQry v2.0+ is the recommended version for 2026. It includes enhanced IPv6 support and better protocol detection.

Basic Syntax

portqry -n [target] -p [protocol] -e [port]

Port States Explained

State Meaning
LISTENING Port is open and accepting connections
NOT LISTENING Port is closed or no service running
FILTERED Port is blocked by firewall or no response

‚öôÔ∏è PortQry Usage Guide

Common Parameters

Parameter Description
-n Target hostname or IP address
-p Protocol: TCP, UDP, or BOTH
-e Single port to query
-r Range of ports (e.g., 100:200)
-o Multiple specific ports (e.g., 80,443,3389)
-l Output to log file
-y Overwrite existing log file

Example Commands

Check single port:

portqry -n server.domain.com -p tcp -e 80

Check multiple ports:

portqry -n 192.168.1.100 -p tcp -o 80,443,3389,8080

Check port range:

portqry -n 10.0.0.50 -p tcp -r 1:1024

Check UDP port:

portqry -n dns-server.local -p udp -e 53

Check both TCP and UDP:

portqry -n mailserver.com -p both -e 25

Save to log file:

portqry -n 192.168.1.10 -p tcp -e 443 -l portqry.log -y

üì° What is Telnet?

Telnet is a network protocol and command-line tool used to establish remote connections and test port connectivity. While it's not secure for remote administration (SSH is preferred), it remains valuable for network troubleshooting in 2026.

Why Use Telnet for Port Testing?

  • Quick way to test if a TCP port is open
  • Built into most operating systems
  • Immediate feedback on connection status
  • Can send raw commands to services
  • Lightweight and fast

Enabling Telnet Client (Windows)

# Open PowerShell as Administrator
dism /online /Enable-Feature /FeatureName:TelnetClient

# Or use Windows Features GUI
Control Panel > Programs > Turn Windows features on or off
‚òë Telnet Client

Checking if Telnet is Installed

telnet
# If installed, you'll see: Welcome to Microsoft Telnet Client
‚ö†Ô∏è Security Warning: Never use Telnet for remote login in production. All data, including passwords, is sent in plain text. Use SSH instead for secure connections.

üîç Troubleshooting Guide

Common Issues & Solutions

Issue: "Telnet is not recognized"

# Solution: Enable Telnet Client
dism /online /Enable-Feature /FeatureName:TelnetClient

# Or install via Windows Features
Settings > Apps > Optional Features > Add Telnet Client

Issue: PortQry shows "FILTERED"

Possible Causes:
‚Ä¢ Firewall blocking traffic
‚Ä¢ Network ACL restrictions
‚Ä¢ Router blocking packets
‚Ä¢ Host-based firewall active

Solutions:
1. Check Windows Firewall rules
2. Verify network firewall policies
3. Confirm service is running
4. Check antivirus/security software

Issue: Telnet connects but shows blank screen

‚úÖ This is NORMAL! A blank screen means the port is OPEN. The service is waiting for your input. Try typing a command specific to that service or press Ctrl+] to exit.

Issue: "Connection timed out"

# Possible reasons:
1. Host is offline or unreachable
2. Firewall dropping packets (no response)
3. Network routing issue
4. Service not running on target port

# Troubleshoot with ping first:
ping hostname.com

# Then check with PortQry:
portqry -n hostname.com -p tcp -e [port]

Issue: UDP ports show as "LISTENING or FILTERED"

üí° Explanation: UDP is connectionless, so PortQry can't definitively determine if a UDP port is listening without protocol-specific queries. Use the -nr flag for more aggressive testing.

Issue: Getting different results from different tools

Tool What It Tests Best For
Telnet TCP connection attempt Quick TCP port checks
PortQry Protocol-aware testing Detailed diagnostics
Nmap Advanced port scanning Security audits
Test-NetConnection PowerShell native test Windows automation

Best Practices for 2026

‚ö†Ô∏è Common Security Violations Found in Audits

TOP 10 PORT SECURITY MISTAKES (2026):

1. RDP (3389) Open to Internet
Test: telnet public-ip 3389
Fix: Block at firewall, use VPN

2. Database Ports Exposed Externally
Test: portqry -n public-ip -p tcp -e 1433,3306,5432
Fix: Internal network only

3. Telnet Still Enabled
Test: telnet server 23
Fix: Disable telnet service completely

4. Unencrypted FTP
Test: telnet server 21
Fix: Replace with SFTP or FTPS

5. SMB (445) Open to Internet
Test: portqry -n public-ip -p tcp -e 445
Fix: Block at perimeter firewall

6. Default Ports on Custom Apps
Issue: Everyone scans 8080, 8443
Fix: Use non-standard ports + obscurity

7. Unnecessary Services Running
Test: portqry -n server -p tcp -r 1:10000
Fix: Disable unused services

8. Missing Firewall Rules Between Zones
Test: Port accessible from DMZ to internal
Fix: Implement zone-based security

9. VPN Ports Blocked Accidentally
Test: portqry -n vpn -p udp -e 500,4500
Fix: Allow VPN protocols (IPSec, OpenVPN)

10. Monitoring/Management Ports Exposed
Test: Ports 9090, 8443, 10000 from internet
Fix: Internal network or VPN only
